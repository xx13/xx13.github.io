Pleroma, Visual Concept

Last update: April 22, 2019.

Visual Concept

There are 2 things I kept in mind when designing the visual concept for Pleroma. The first one is sticking to key decisions, the second one is keeping the new theme related to the current visual style.

There were some minor changes to the structure as I had more time to work on the details. Navigation at the very top and sub-nav right below it were merged into one section, sorting options for the feed were removed as only 'show latest first' make sense. The other impotant change is adding 'Add New Entry' action available on any screen.

You should refer to the following screens as the general direction for design rather than a polished and finished app screens.

Explore

Default feed inlcudes all content types that are pre-sorted — newest first and your feed or public timeline or your server by default (it depends on who are the users, what way we expect them to work with the app). For now I used Public Timeline as default feed so users can Explore things more w/o following/subscribing others.

Content type filters are available at the top and will work for User's Feed view as well.

Posts/entries/card cover the entire screen and flow covering all available space. Quick actions are available on the go.

We might consider adding custom colorful backgrounds but it may result in poor readability as the screen will be filled with screaming colors. The other way to introduce a little contrast is to add branded colroful background for trending / popular / featured posts only.

It's very easy to make the entire platform YouTube-ish by simply applying Videos filter at the very top. Search and feed options will work together with content filters so users can get the content they want.

The layout allows to display videos with heading / text / heading + text options. Based on users' feedback it might be a nice touch hide text description (or add 'Hide text' option) so the focus will be entirely on videos.

User's Profile

The layout and all sorting options are the same as for Explore feed. The only difference is a User's section that contains an overview, bio (optional), acitons (Follow, Report, Block, etc) and visual elements (userpic and custom background).

I highligted 'Follow' as the main action. Meanwhile other actions were grouped together and put under 'More' icon. So they are still available but no require too much attention from users.

The layout will work for users w/o bio information as well.
